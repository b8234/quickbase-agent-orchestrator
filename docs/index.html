Quickbase Agent Orchestrator

Serverless AI workflow enabling conversational access to Quickbase data

Overview

This project demonstrates how to combine Amazon Bedrock, AWS Lambda, API Gateway, S3, Transcribe, Slack, and Quickbase to build a distributed, event-driven, conversational workflow system. It is a lightweight reference implementation for building natural-language interfaces over enterprise data.

What This Project Demonstrates

  • Distributed workflow orchestration using AWS serverless components
  • AI-driven reasoning and action routing via Amazon Bedrock Agents
  • Voice and text multimodal input with Streamlit + Transcribe
  • Secure Quickbase access with a field-level allowlist governance layer
  • Enterprise patterns for exports, Slack notifications, and record validation

System Components

Streamlit Frontend

  • Supports voice and text input
  • Displays AI responses, tables, and export links
  • Communicates with backend via API Gateway

Lambda Frontend

  • Handles UI requests
  • Transcribes audio through Amazon Transcribe
  • Invokes Amazon Bedrock Agent for reasoning and routing

Bedrock Agent

  • Routes user intent to backend Lambda
  • Coordinates multi-step reasoning

Lambda Backend

  • Queries Quickbase records using a governed allowlist
  • Summarizes data and formats structured responses
  • Exports CSV/JSON files to S3 with pre-signed URLs
  • Sends Slack notifications for reporting workflows

Governance Layer

A field-level allowlist ensures safe, controlled access to Quickbase tables and fields, enabling secure AI-driven workflows in enterprise environments.

Demo Mode

Set DEMO_MODE=true inside .env to run the Streamlit UI locally without AWS or Quickbase credentials.
